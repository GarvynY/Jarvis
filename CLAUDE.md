# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

**PythonClaw** is a Python-native autonomous AI agent framework. It supports multiple LLM providers (DeepSeek, Grok, Claude, Gemini, Kimi, GLM, or any OpenAI-compatible API), multi-channel messaging (Telegram, Discord, WhatsApp, Web, CLI), persistent memory, hybrid RAG, a three-tier skill system, and daemon mode.

## Commands

### Development Setup
```bash
python -m venv .venv && source .venv/bin/activate
pip install -e .
```

### Testing
```bash
pytest tests/ -v                           # Full test suite
pytest tests/test_skills.py -v             # Single file
pytest tests/test_soul.py::TestPersona -v  # Single class
```

### Linting
```bash
ruff check pythonclaw/
```

### CLI
```bash
pythonclaw onboard                         # Interactive setup wizard
pythonclaw start                           # Start daemon (web UI on port 7788)
pythonclaw start -f                        # Foreground mode
pythonclaw chat                            # Interactive CLI chat
pythonclaw skill search <query>            # Search marketplace
pythonclaw stop / status                   # Daemon control
```

## Architecture

### Core Reasoning Loop
`pythonclaw/core/agent.py` — main reasoning loop, manages conversation history, dispatches tool calls, triggers compaction. Does **not** manage session lifecycle, persistence, or I/O channels.

### Session Management
- `session_manager.py` — central registry of `session_id → Agent`; per-session `asyncio.Lock` + global `asyncio.Semaphore` (default 4) prevent message interleaving and cap concurrency.
- `core/session_store.py` — disk-backed persistence of agent state across restarts.

### LLM Provider Abstraction (`core/llm/`)
- `base.py` — abstract `LLMProvider` class all providers implement.
- `openai_compatible.py` — covers DeepSeek, Grok, Kimi, GLM.
- `anthropic_client.py` / `gemini_client.py` — native clients for Claude and Gemini.
- To add a provider: subclass `LLMProvider`, add factory logic in `main.py:_build_provider()`.

### Memory System (`core/memory/`)
- Markdown-based long-term memory in `~/.pythonclaw/context/memory/`.
- `MEMORY.md` — curated latest key-value pairs; `YYYY-MM-DD.md` — append-only daily logs.
- Recall uses hybrid retrieval (BM25 + optional dense embeddings).
- Per-group isolation optional (each group/channel gets its own memory dir).

### Hybrid RAG Pipeline (`core/knowledge/`, `core/retrieval/`)
Flow: `Query → BM25 (sparse) + Dense Embeddings → RRF Fusion → LLM Re-ranker → Top-K`
- `knowledge/rag.py` — `KnowledgeRAG`, loads corpus from files/dirs.
- `retrieval/retriever.py` — `HybridRetriever` orchestrates sparse + dense + fusion + reranking.

### Three-Tier Skill System (`core/skill_loader.py`, `core/skillhub.py`)
- **L1 (Metadata)**: YAML frontmatter loaded at startup.
- **L2 (Instructions)**: Full `SKILL.md` loaded on agent activation.
- **L3 (Resources)**: Bundled scripts, schemas, data files.
- `SkillHub` client connects to ClawHub marketplace (13K+ community skills).
- To add a skill: create `pythonclaw/templates/skills/<category>/<skill_name>/SKILL.md`.

### Tools (`core/tools.py`)
Primitive tools (bash, Python, file ops), memory tools, knowledge tools, skill tools (dynamically built per session), cron tools, and Tavily web search. Tool results are logged to `context/logs/history_detail.jsonl`.

### Context Compaction (`core/compaction.py`)
Auto-compacts conversation history when token threshold is hit (configurable via `agent.autoCompactThreshold` in config). Keeps N recent messages intact and summarizes older ones.

### Web Dashboard (`web/app.py`)
FastAPI app serving real-time chat, skill catalog, marketplace browser, and config editor at `localhost:7788`. Includes optional Deepgram STT integration.

### Multi-Channel Support (`channels/`, `server.py`, `daemon.py`)
Each channel (`telegram_bot.py`, `discord_bot.py`, `whatsapp_bot.py`) uses `SessionManager` for lifecycle. `server.py` orchestrates all channels async. `daemon.py` handles PID-based background process management.

## Configuration

Config lives at `~/.pythonclaw/pythonclaw.json` (generated by `pythonclaw onboard`). See `pythonclaw.example.json` for all available fields. Environment variables override JSON (e.g., `DEEPSEEK_API_KEY`, `ANTHROPIC_API_KEY`).

Key config sections: `llm` (provider + API keys), `channels` (Telegram/Discord/WhatsApp tokens), `isolation.perGroup` (per-session memory), `concurrency.maxAgents`, `agent.autoCompactThreshold`.
