## Agent 不是一次性问答

如果一个 Agent 要真正进入个人工作流，它需要理解长期偏好、重复任务和上下文边界。

但是记忆不是越多越好。Jarvis 里的经验是：

- 用户偏好需要结构化
- 敏感上下文需要隔离
- 研究链路只能读取安全摘要
- 反馈按钮需要反哺个性化策略

<span class="color-blue">产品体验上的“聪明”</span>，很多时候来自稳定的上下文管理，而不是单次模型能力。

## 一个产品原则

> Agent 应该记住有用的稳定偏好，而不是无边界地保存一切。
